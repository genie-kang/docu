# 로깅 레벨 변경
---

에이전트의 로그 파일은 문제를 해결하기 위해 유용한 정보를 제공합니다. 문제 해결 시에는 상세 로깅을 활성화하고 문제를 재현하여 로그 파일에 관련 정보를 기록하십시오.<br />
상세 로깅은 로그 파일의 용량을 늘리므로 문제 해결 목적으로만 사용해야 합니다. 에이전트 로그 파일은 관리 콘솔에 30일 동안 보관됩니다.

## Step
- 관리 콘솔의 메뉴 표시줄에서 자산 > 장치를 클릭합니다.
- 기기를 클릭하십시오.
- 다음 중 하나를 수행하십시오.
    - 로그 수준을 변경하려면 에이전트 로깅 수준 드롭다운 목록에서 로그 수준을 클릭합니다.
    - 에이전트 로그 파일을 가져오려면 '위협 및 활동' 아래의 '에이전트 로그'탭에서 '현재 로그 파일 업로드'를 클릭하십시오,<br /> 이 옵션을 장치가 온라인 상태인 경우에만 사용할 수 있습니다.

## 로컬 장치에서 로깅 활성화

### 1. 윈도우에서 로깅 활성화
- 시스템 트레이에서 에이전트 아이콘을 우클릭하여 종료합니다.

- cmd를 관리자 권한으로 실행합니다.

- 다음 명령어를 실행하십시오. <br /> `cd C:\Program Files\Cylance\Desktop`<br />`CylanceUI.exe -a`

- 시스템 트레이에서 에이전트 아이콘을 우클릭한 후 메뉴에서 로깅 > 모두를 클릭하십시오.

### 2. macOS에서 로깅 활성화
- 현재 실행 중인 사용자 인터페이스를 종료합니다.

- 터미널에서 다음 명령어를 실행하십시오.<br />`sudo /Applications/Cylance/CylancePROTECTUI.app\`<br />`Contents/MacOS/CylancePROTECTUI.-a`

- 시스템 트레이에서 에이전트 아이콘을 우클릭한 후 로깅 > 모두를 선택하십시오.

### 3. Linux에서 로깅 활성화
- 다음 명령어를 실행하여 에이전트의 현재 로깅 수준을 확인하십시오.<br />`/opt/cylance/desktop/cylance -l`
&nbsp;

- 다음 명령어를 실행하여 로깅 수준을 변경하십시오.<br />`/opt/cylance/desktop/cylance -L <level>`
    - **level** 의 값은 다음 중 하나입니다.
        - **0** : Error
        - **1** : Warning
        - **2** : Information
        - **3** : Verbose
-  로깅 수준을 '자세히'로 설정하려면 다음 명령어를 실행하십시오.<br />`/opt/cylance/desktop/cylance -L 3`
